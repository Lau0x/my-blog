# v1.2.0 · 站点可配置化 + 自定义页面 + 导航定制

> 发布日期：2026-04-18 · [GitHub Release 页面](https://github.com/Lau0x/my-blog/releases/tag/v1.2.0)

第二个版本。核心是让**非技术用户也能在 admin 里把博客玩出花**——不用改代码就能定义站点名、建独立页面、定制导航栏。

## ✨ 新功能

### 🎨 全站可配置化
- **站点配置** (Single Type `SiteConfig`) — admin 里改 `siteName` / `tagline` / `footerHTML` / OG 图 / themeColor，前台立即生效。footerHTML 支持 `{{year}}` 模板自动刷新版权年份。
- **完全自定义导航栏**（`SiteConfig.navItems` Repeatable Component）— 填 `label + path + order` 覆盖默认导航。支持内链（`/tags`）、内建路由（`/`）、**外链**（`https://github.com/xxx` 自动新窗口打开）。不配置则回退默认（首页 + 标签 + showInNav 的 CustomPages）。

### 📄 自定义页面
- **CustomPage Collection Type** — 建 `/about` / `/vps` / `/friends` 这种独立页，admin 里 `Create new entry` → 填 slug/title → 发布。
- **双渲染模式**：
  - **Markdown** — 和文章一样的管道，自动标题 + `.prose` 正文样式
  - **Raw HTML**（`htmlBody` 字段，richtext 大编辑器）— **完全自由**，支持 `<style>` + `<script>` + flex 布局。适合做 VPS 推广、简历卡、联盟页。`htmlBody` 非空时自动接管，隐藏自动标题和 `.prose` 样式。
- CustomPage 勾选 `showInNav`（navItems 为空时生效）即可进导航。

### 🏷️ 标签聚合页 `/tags`
- 展示全部 tag + 每个 tag 下的文章列表
- 按文章数倒序排，一眼看清博客内容分布

### 🔼 回到顶部按钮
- 滚动 >300px 后右下角浮出，点击 smooth scroll 到顶
- 移动端自适应尺寸

### 🔗 外链自动新窗口
- 文章正文 / CustomPage / 导航栏里所有 `http(s)` 外链**自动 `target="_blank"` + `rel="noopener noreferrer"`**
- 锚点（`#xxx`）、内链、`mailto:` 协议不受影响
- 用户在 htmlBody 里自己写了 `target="_self"` 不会被覆盖

## 🛠️ 工程改进

- **Docker 镜像 tag 命名规范化** — git tag `v1.2.0` ↔ docker tag `v1.2.0`（之前 CI 会剥 `v` 前缀）。现在 `IMAGE_TAG=v1.2.0` 和 Git Release 名完全一致，pin 版本更直观。v1.1.0 的旧命名 `1.1.0` / `1.1` 已清理。
- **Pin 版本策略文档** — `docs/deployment/upgrade.md` 加了完整的 "严肃生产 pin 版本" 操作手册 + 三种 tag 对比表 + `docker manifest inspect` 自助验证 recipe。
- **Release notes 归档** — `docs/release-notes/` 目录入库，v1.1.0 / v1.2.0 均归档。

## 📋 升级指南（非破坏性 + 3 步配置）

```bash
# Step 1. 拉新版镜像 + 重启
ssh <你的博客服务器>
cd /path/to/blog
vim .env                    # IMAGE_TAG=v1.2.0（推荐 pin 到具体版本）
./scripts/upgrade.sh        # 会自动备份 + pull + 重启
```

```
Step 2. admin 开新 content-type 的 Public 权限（必做，否则前端拿 403）

Settings → Users & Permissions → Roles → Public → 勾：
  - Site-config: find
  - Custom-page: find + findOne

Step 3. （可选）admin 里填 SiteConfig

Content Manager → 站点配置 → 改 siteName / tagline / footerHTML
  → 首页 header + footer 立即生效

Content Manager → 自定义页面 → Create new entry（slug=about 等）

（可选）回到 SiteConfig → navItems 自定义导航栏
```

**数据层面完全兼容**：v1.1.0 的 article / user / 广告配置 / Tag 组件不受影响。Strapi 启动时会自动 ALTER TABLE 加 `site_configs` / `custom_pages` / `components_nav_nav_items` 等新表，无需手动 migration。
